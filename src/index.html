Customización de registro de contratos leasing en un sistema ERP

ERP Sector financiero

En este proyecto implementé una customización de un módulo de contratos dentro de un ERP, realizando, modificando y creando extensiones para la creación de nuevos contratos de un cliente de leasing del mercado suizo. Me enfoqué en dos tipos principales de contratos: Leasingvertrag (contrato de leasing), Mietvertrag (contrato de alquiler) La personalización implicó desarrollar un sistema flexible y robusto capaz de manejar diferentes cálculos de interés, ajustándose a las variaciones en las tasas moratorias, factores de riesgo y otros elementos críticos en los cálculos financieros definidos por el usuario. Para ello se solicitó la modificación de la interface gráfica ( GUI) así como algoritmos internos. La tecnología utilizada fue VB NET.

Sistema base para gestión de fideicomisos de de cobertura

En este proyecto, actualmente estoy desarrollando una aplicación de gestión de solicitudes de empresas por créditos de cobertura a una Corporación Financiera de Desarrollo del gobierno peruano. El proceso contempla el proceso de admisión, honramiento de la deuda y la generación de órdenes de giro y notificaciones a los beneficiarios y a las entidades financieras. El objetivo además es implementar un software base y que se pueda customizar a futuro según los nuevos programas de financiamiento promovidas por el gobierno de turno.

Aplicativo web dotación de recursos humanos

Eficiencia operativa SSD

En este proyecto implementé un módulo en una plataforma SPA para mantenimiento de personal y requerimientos registrados por jefes autorizados y enviado al personal de recursos humanos y automatización los procesos de dotación dentro del área de control y suministros. La problemática era que el área de operaciones tenía un número de personal que necesitaba, el área de recursos humanos tenía otro número diferente y TI debía solucionar el problema e “igualar” ambos números. La implementación consideró algunos cálculos de ambas áreas y finalmente se llegó a igualar ese número y a armonizar el trabajo entre esas dos áreas.

Utilicé tecnología Angular Material y en el backend WCF.

Módulo de requerimientos de activos y suministros

En este proyecto implementé un módulo de mantenimiento para registrar requerimientos de activos requeridos y utilizados por las tiendas y su notificación respectiva al área de control y suministros. La problemática era que no se contaba con un sistema y todo se enviaba por correo, dificultando los procesos de atención. Para facilitar el uso se mostró un catálogo de activos con sus respectivas imágenes, descripciones y precios unitarios.

Notificación por Whatsapp (tarea programada)

En este proyecto, implementé alerta con una tarea programada que se comunique con el API de WhatsApp a los gerentes zonales y otros responsables del área de operaciones de una empresa de sector retail donde muestre un resumen de las ventas del día. De esta forma se puede dar seguimiento a qué sector (grupo de las tiendas) cumple con las metas diarias de ventas. Las tecnologías utilizadas en este caso fue el API mediante el uso de Meta for Developers. Para el API se utilizó Express (TS) y una tarea programada con C#.

Mantenedor de asignación de zonas y actividades de contratistas

En este proyecto implementé un mantenedor de asignación de capacidades y zonificación para subcontratistas en el ámbito de las conexiones eléctricas. Mi experiencia en estas tecnologías ha sido fundamental para crear una interfaz de usuario intuitiva y una arquitectura de backend robusta, lo que permite a los usuarios establecer y gestionar de manera efectiva las capacidades y áreas de operación específicas de cada subcontratista. La integración de estas tecnologías asegura una asignación eficiente y precisa de los subcontratistas, optimizando la ejecución de los proyectos y mejorando la gestión de recursos. Este sistema ha sido desarrollado utilizando tecnologías jQuery, Bootstrap 4 y .NET MVC para aplicaciones web.
